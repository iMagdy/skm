# Sponsoring skm

`skm` is an open source CLI for sharing, distributing, and installing agent skills.

The goal of `skm` is to make skills portable, discoverable, installable, and reusable across AI agents and developer workflows.

Sponsorship helps keep the project maintained, independent, and useful for the wider community.

## Why sponsor skm?

Sponsor `skm` if you believe agent skills should be easy to package, share, install, and reuse.

Your sponsorship helps fund:

- Continued development and maintenance
- Cross-platform release binaries for macOS, Windows, and Linux
- Better documentation and examples
- Registry and manifest design
- Security improvements
- Compatibility testing
- Community support
- Future support for agents, not only skills

## Sponsorship options

Individuals and companies can sponsor the project through the official sponsor links in this repository.

If you are a company interested in larger sponsorship, support agreements, roadmap discussions, or partnership opportunities, please contact the maintainer through the official project channels.

## Sponsor recognition

Sponsors may be recognized in project materials, depending on their sponsorship tier and preference.

Possible recognition includes:

- Listing in this file
- Mention in release notes
- Logo placement in the project README or website
- Early visibility into roadmap discussions
- Priority consideration for issues related to supported use cases

Sponsor recognition does not mean control over the project.

## Project independence

Sponsorship supports the project, but it does not buy ownership, trademark rights, release rights, or maintainer authority.

The `skm` name, official releases, project direction, and governance remain controlled by the project maintainers.

Sponsors may suggest features, report issues, fund specific work, or support priorities, but maintainers make final project decisions.

## Good sponsorship fit

Sponsorship is especially welcome from:

- Developers using `skm`
- Teams building agent tooling
- Companies building AI developer platforms
- Organizations interested in portable agent skills
- Registry, marketplace, or package ecosystem builders
- Teams that want reliable skill distribution for internal agents

## Funded work

Sponsors may fund general maintenance or specific areas of work.

Examples:

- Improved installer experience
- Stable manifest schema
- Skill validation and security checks
- Registry support
- Better examples and documentation
- CI/release automation
- Agent support
- Enterprise-friendly workflows

If you want to fund a specific feature, please open a discussion first so we can align on scope, expectations, and whether the feature fits the project.

## What sponsorship does not include

Sponsorship does not include:

- Ownership of the project
- Exclusive control over roadmap
- Permission to use the `skm` name as your own product
- Guaranteed feature delivery unless separately agreed
- Private support unless explicitly agreed
- Endorsement by the `skm` project
- Permission to publish official-looking `skm` releases

For brand and trademark rules, see `TRADEMARK.md`.

## Companies

Company sponsorship is welcome.

Companies can sponsor `skm` to support:

- Open source sustainability
- Tooling they depend on
- Features useful to their teams
- Security and reliability improvements
- Long-term ecosystem health

For larger sponsorships, please contact the maintainer to discuss the best path.

## Thank you

Every sponsorship helps keep `skm` moving forward.

Thank you for supporting open source infrastructure for agent skills.
