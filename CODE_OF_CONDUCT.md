# Code of Conduct

`skm` is a small open source project, and we want contribution to feel clear,
direct, and respectful.

## Our Standards

Expected behavior includes:

- Be welcoming and constructive.
- Assume good intent while still being precise about problems.
- Focus criticism on ideas, code, behavior, and project fit.
- Respect maintainers' time and contributors' boundaries.
- Accept moderation decisions and move on.

Unacceptable behavior includes:

- Harassment, threats, insults, or personal attacks.
- Sexualized language or imagery.
- Deliberate intimidation, stalking, or sustained disruption.
- Publishing private information without explicit permission.
- Repeatedly ignoring maintainer guidance after a clear decision.

## Scope

This code of conduct applies in project spaces, including GitHub issues, pull
requests, discussions, releases, and official community channels.

## Enforcement

Maintainers may remove comments, close issues or pull requests, block users, or
take other action needed to keep the project healthy.

Report conduct concerns privately through the repository owner profile. For
security vulnerabilities, follow `SECURITY.md` instead.
