# Data Model: Test Coverage & Integration Tests

**Feature**: 004-integ-tests-coverage
**Date**: Sat May 30 2026

## Entities

### TestFixture

Represents a pinned remote repository used as test input.

| Field | Type | Description |
|-------|------|-------------|
| name | String | Human-readable name (e.g., "awesome-copilot") |
| url | String | Git clone URL |
| commit_sha | String | Pinned commit SHA for reproducibility |
| has_manifest | bool | Whether repo contains skills.json |
| skills_dir | Option<String> | Path to skills/ directory if present |

**Relationships**: Referenced by integration tests to clone and validate against.

### CoverageReport

Output from cargo-tarpaulin showing coverage metrics.

| Field | Type | Description |
|-------|------|-------------|
| line_coverage | f64 | Percentage of lines covered |
| function_coverage | f64 | Percentage of functions covered |
| branch_coverage | f64 | Percentage of branches covered |
| uncovered_lines | Vec<UncoveredLine> | Specific lines not covered |

**Relationships**: Generated by coverage tooling, evaluated against 95% threshold.

### UncoveredLine

Represents a specific line lacking test coverage.

| Field | Type | Description |
|-------|------|-------------|
| file | String | Source file path |
| line_start | u32 | Starting line number |
| line_end | u32 | Ending line number |
| function_name | Option<String> | Containing function if applicable |

**Relationships**: Child of CoverageReport, used to identify what needs tests.

### IntegrationTestResult

Outcome of running an integration test.

| Field | Type | Description |
|-------|------|-------------|
| test_name | String | Test function name |
| fixture | TestFixture | Repository used |
| status | TestStatus | Pass/Fail/Skip/Error |
| duration_ms | u64 | Execution time |
| retry_count | u32 | Number of retries attempted |
| error_message | Option<String> | Failure details if applicable |

**Relationships**: Generated during test execution, evaluated for CI gate.

### TestStatus

Enum representing test execution state.

| Variant | Description |
|---------|-------------|
| Pass | Test completed successfully |
| Fail | Test failed after all retries |
| Skip | Test skipped (network unavailable, ignored) |
| Error | Test encountered unexpected error |

## State Transitions

### Integration Test Lifecycle

```
Created → Running → Pass
                  → Fail (after 3 retries) → Retrying (up to 3 times)
                  → Skip (if #[ignore] and not run with --ignored)
                  → Error (unexpected)
```

### Coverage Gate Lifecycle

```
Measured → Below 95% → Blocked (CI fails)
        → At/Above 95% → Passed (CI succeeds)
```

## Validation Rules

- commit_sha must be valid hex string (40 characters)
- line_coverage must be between 0.0 and 100.0
- retry_count must be between 0 and 3
- test_name must be non-empty
- fixture URL must be valid git URL

## File Locations

| Entity | File | Format |
|--------|------|--------|
| TestFixture | `tests/integration/*.rs` | Rust constants |
| CoverageReport | CI output | LCOV/XML |
| IntegrationTestResult | CI output | JUnit XML |
